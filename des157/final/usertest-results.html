User Test Results

Please test the scrolling to see if it will follow. Does it scroll and follow? Is this a good idea?

User response: I don’t see anything “following” when
I scroll. I am not sure what I am
supposed to be looking for. I am not sure
if it’s a good idea or not because I
am not sure what is supposed to be following?
Is there supposed to be a sticky element or
a fixed element? I like the concept of a
website that is a cherry blossom tree with blossoms
being the clickable elements though.

Please test tapping on an icon. Do you think this is helpful? Do you think it can be improved?

User response: When hovering over the blossom
icons some descriptive text appear describing what
I will see when I click. I like this as a
way of communicating that the blossom is clickable
but it would be nice if the blossom also
scaled up a bit; like scale: 1.1 on hover
would be nice to better communicate the interactivity.
I also would have liked to see non-distorted images.
The stretching of the photos makes them a lot
less pleasing to look at than if they were a
bit smaller but not stretched.

Please test the sound if it is working. Is the sound needed or make a difference?

User response: The sound added a nice atmospheric touch,
but it was unclear what it’s purpose was until
I clicked on it. Then I was like, “ah okay,
it’s atmospheric.” Something to indicate the
purpose of the sound to the user would have
been nice so I could decide whether I want to
click before I click.

User Test Results Analysis

Observation 1

The user had trouble understanding, on the website,
what I meant by scrolling and following an element
because I was unable to attach and element with
the scrolling effect. Of course, the user was
not able to tell what would be a good idea
or not because there was nothing they could
reference. They did ask if there was suppose
to be a sticky element or a fixed element
that was supposed to be followed on the scrolling
within the window. I am glad that the user
recognized that I was going for the cherry blossom
theme and that it was obvious on the screen.

Observation 2

Intentionally, I wanted the user to have some sort
of feedback when they hover over an interactive element.
I understand that my approach may not be the
best approach to having some sort of user interaction,
but it is still a form of interactive feedback to
the user. It was suggested that I up scaled the
cherry blossom image a bit more when hovering over
it, but I believe that the way I designed it will
not allow me to do that so easily. With the images,
I believe that they were distorted because of meeting
the image requirement of keeping images under 200KB.
That is why I had to compress the image and lower
the quality to meet these conditions. I also did
not really figure out how to unstretch the image.

Observation 3

This was a pretty straightforward question to ask about
the sound and if it really made any big difference
to the experience. What the user said is that
it added a nice atmospheric touch to the website
but it was unclear what it's purpose was
until the user clicked on the play button.
The user seemed to have needed to click the
sound before the user realized what it was
for. What I had intended here was that the sound
be played automatically and have cherry blossoms
fly across the screen to simulate something
blowing in the wind. This was something I want
the user to experience when they enter the site.
To have something similar to a windy, outside experience.
